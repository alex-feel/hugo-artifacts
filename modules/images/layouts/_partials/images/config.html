{{/* images/config.html -- INTERNAL (resolver tier). Purpose: resolves the module's entire configuration through the four-tier cascade, so every surface renders from ONE consistent options dict and no downstream partial re-derives a value. Precedence, highest first: call-site args > page front matter (img: map) > site params ([params.img]) > data/images/defaults.toml. Presence wins at every tier (isset, not truthiness), so an explicit false, 0, or "" overrides the tier below it. Nested sections (quality, process_defaults, placeholder, lightbox, theme, remote, hook, gallery) cascade per KEY within the section, so a page can override quality.webp alone without restating the section. The namespace is `img` (NOT `images`): Hugo's embedded Open Graph and Twitter Cards templates and common front-matter conventions already claim `images` as an ARRAY, which cannot coexist with a map at the same key. Because shortcode named parameters and hook Markdown attributes deliver STRINGS only, every typed key is normalized here, at one place, with one uniform contract per type class: - bool: truthy iff "true"/"1"/"yes"/"on" (case-insensitive); native bools pass through. - int: integer cast after stringifying; garbage warns once and the tier below wins. - dimension: images/resolve/dims.html (int, float, "1920px" strings). - int slice: native slices pass through; "480,800" strings comma-split; garbage tokens warn once and drop; an all-garbage value collapses to the tier below; an explicitly empty value is honored as empty. - token slice (formats): allowed tokens 'webp' and 'avif'; unknown tokens warn once and drop. - enum: unknown tokens warn once and fall back to the feature default. Expected context (dict): - .page (Page) The current page (already validated by the caller). - .args (map) Call-site arguments (possibly empty), flat names per the README Parameters table. `fetchpriority` is deliberately accepted at the call tier ONLY (a site-wide deprioritized-images default must be impossible to configure).
