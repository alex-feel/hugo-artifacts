{{/* images/resolve/dims.html -- PURE (resolver tier). Purpose: normalizes raw width/height values into integers. Shortcode named parameters and Markdown attributes deliver strings, front matter may hold ints or floats, and authors write "1920px"-style values by hand; this partial accepts all of those shapes at ONE place so no caller ever feeds a non-numeric string into an integer cast (a build-crash class this module designs out). Digit strings parse through images/lib/int.html, so a leading zero reads as decimal (never octal) and an oversized digit string degrades to unparseable instead of overflowing the cast. Fractional strings carry the same nine-digit cap on their integer part, so an oversized fractional value degrades to unparseable too instead of overflowing the float cast. Context (dict): - .width (any, optional) Raw width value: int, float, "1920", "1920px". - .height (any, optional) Raw height value, same shapes. Return value: dict - width (int) Parsed width, 0 when absent or unparseable. - height (int) Parsed height, 0 when absent or unparseable. - ok (bool) false when any SUPPLIED value failed to parse (the caller warns once and ignores the failed value); true otherwise. One terminal return. */}} {{- $ctx := . -}} {{- $out := dict "width" 0 "height" 0 "ok" true -}} {{- range $k := slice "width" "height" -}} {{- $raw := index $ctx $k -}} {{- if ne $raw nil -}} {{- $s := lower (trim (printf "%v" $raw) " \t\n\r") -}} {{- $s = strings.TrimSuffix "px" $s -}} {{- $n := -1 -}} {{- if findRE `^\d+$` $s -}} {{- $n = partial "images/lib/int.html" $s -}} {{- end -}} {{- if ge $n 0 -}} {{- $out = merge $out (dict $k $n) -}} {{- else if findRE `^\d{1,9}\.\d+$` $s -}} {{- $out = merge $out (dict $k (int (float $s))) -}} {{- else if ne $s "" -}} {{- $out = merge $out (dict "ok" false) -}} {{- end -}} {{- end -}} {{- end -}} {{- return $out -}}
