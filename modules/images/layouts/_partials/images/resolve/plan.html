{{/* images/resolve/plan.html -- PURE (resolver tier). Purpose: turns a resolved, processable source plus the cascaded options into a derivative plan: post-orientation source dimensions, the effective candidate list (no-upscale clamp, dedupe, density cap), the gated format chain list with per-chain spec suffixes, the sizes attribute value, and the lightbox cap geometry. resolve/srcset.html executes this plan; nothing in this partial touches the image pipeline, so the plan itself is free. Width-candidate rules (constrained/full layouts, w-descriptors): 1. Start from the configured ladder, sorted ascending, deduplicated. 2. Drop every entry STRICTLY GREATER than min(srcW, cap) -- never upscale, never exceed the density cap; an entry exactly EQUAL to the limit stays. 3. Append min(srcW, cap) itself as the top candidate when it is BOTH smaller than the configured ladder's largest entry AND not yet present. 4. cap = max_density * width for an explicit width in constrained layout (2x fidelity cap by default); unlimited otherwise. responsive=false collapses the plan to one candidate: min(srcW, width or ladder top). Fixed layout (x-descriptors): the effective density list is the configured densities filtered to entries <= max_density, with max_density appended when it exceeds the list maximum; each density d is emitted only when width*d <= srcW. A height-only fixed call works: the height-only resize conversion below runs BEFORE the density build, deriving the equivalent width from the source aspect ratio (the calling surfaces reject only the no-width-AND-no-height case). When even 1x exceeds the source width the plan degrades to the bare source width and sets fixedOverflow (the caller warns once). Post-orientation dimensions: when the resource supports metadata, EXIF orientation values 5-8 swap the raw .Width/.Height (raw dimensions are pre-orientation truth); every derivative chains images.AutoOrient, so derivative dimensions are post-orientation by construction. AVIF gate: avif chains require Hugo 0.163.0+ (per-image AVIF quality in spec strings landed there); below the gate the token is dropped and avifGated is set (the caller warns once). Probing avif with try is forbidden: a failed avif .Process poisons the warm cache with a 0-byte entry. Context (dict): - .source (dict) resolve/source.html result (processable). - .opts (dict) The cascaded options from images/config.html. - .overrides (dict, optional) Per-call plan overrides: width, height, widths, process, ratio (h/w float, e.g. from a gallery crop). - .eager (bool) Eager/priority loading (suppresses the sizes auto prefix). Return value: dict - srcW, srcH (int) Post-orientation source dimensions. - fixed (bool) layout == "fixed". - candidates (slice) Ascending {w (int), desc (string "640w"/"2x")}. - top (int) The largest candidate width. - chains (slice) Ordered {format, mime, suffix, fallback (bool)}; modern chains first (avif before webp), the original-format fallback chain last. - hasModern (bool) At least one modern chain differs from the fallback. - process (string) resize | fit | fill | crop. - ratio (float) Height/width ratio for two-dimension specs, 0 for width-only specs. - sizes (string) Final sizes attribute value ("" for fixed). - width, height (int) The effective explicit dimensions (0 = unset). - avifGated (bool) avif was requested but dropped below the gate. - fixedOverflow (bool) Fixed width exceeded the source width. - hintDropped (bool) hint was set but a non-WebP chain exists. - compressionDropped (bool) compression was set but a non-WebP chain exists. - lightboxW (int) min(srcW, lightbox width cap). - lightboxSuffix (string) Spec suffix for the capped lightbox derivative (fallback format, resize geometry). One terminal return. */}} {{- $source := .source -}} {{- $opts := .opts -}} {{- $ov := .overrides | default dict -}} {{- $eager := .eager | default false -}} {{- $r := $source.resource -}} {{/* ---- Post-orientation source dimensions ---- */}} {{- $srcW := $r.Width -}} {{- $srcH := $r.Height -}} {{- if $source.withMeta -}} {{- with $r.Meta -}} {{- if in (slice 5 6 7 8) .Orientation -}} {{- $srcW = $r.Height -}} {{- $srcH = $r.Width -}} {{- end -}} {{- end -}} {{- end -}} {{/* ---- Effective geometry ---- */}} {{- $width := $opts.width -}} {{- $height := $opts.height -}} {{- $process := $opts.process -}} {{- $ratio := 0.0 -}} {{- if isset $ov "width" -}}{{- $width = $ov.width -}}{{- end -}} {{- if isset $ov "height" -}}{{- $height = $ov.height -}}{{- end -}} {{- if isset $ov "process" -}}{{- $process = $ov.process -}}{{- end -}} {{- if and (eq $process "resize") (gt $height 0) (eq $width 0) -}} {{/* Height-only resize: proportional by definition, so convert to the equivalent width and keep the single width-driven pipeline. This also serves layout=fixed with only a height: the derived width feeds the density build below exactly as an explicit width would. */}} {{- $width = int (math.Round (div (mul $height $srcW) (float $srcH))) -}} {{- $height = 0 -}} {{- end -}} {{- if and (gt $width 0) (gt $height 0) -}} {{- $ratio = div (float $height) $width -}} {{- end -}} {{- if isset $ov "ratio" -}}{{- $ratio = $ov.ratio -}}{{- end -}} {{/* Defense in depth: fit/fill/crop need a two-dimension spec, so they can only execute when a ratio is set (both dims, or a gallery crop ratio). Filter-crashing one-dimension fit/fill/crop spec. */}} {{- if and (in (slice "fit" "fill" "crop") $process) (le $ratio 0.0) -}} {{- $process = "resize" -}} {{- end -}} {{/* ---- Candidate widths ---- */}} {{- $layout := $opts.layout -}} {{- $fixed := eq $layout "fixed" -}} {{- $candidates := slice -}} {{- $fixedOverflow := false -}} {{- $top := 0 -}} {{- if $fixed -}} {{- $maxd :=
